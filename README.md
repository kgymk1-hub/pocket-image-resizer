# Pocket Image Resizer

Pocket Image Resizer は、Android Chrome などスマホで使いやすいことを優先した Vanilla HTML / CSS / JavaScript 製の軽量 PWA です。React / Vue / Svelte、npm、ビルド環境は使っていません。

## 主な特徴

- **1枚変換**: 1枚の画像をプレビューしながら細かく調整して保存します。
- **一括変換**: 複数画像へ共通設定を適用し、ZIP または連続ダウンロードで保存します。
- **保存形式**: PNG / JPEG / WebP に対応します。JPEG / WebP は品質（低 0.6、標準 0.85、高 0.95）を選べます。PNG では品質設定が不要なため、品質UIは非表示または無効化されます。
- **PWA対応**: `manifest.json`、`service-worker.js`、アイコン一式を維持しています。Service Worker更新検知時は、画面内通知からリロードできます。
- **処理中フィードバック**: 一括変換、ZIP作成、連続ダウンロードでは進捗メッセージを表示し、処理の合間に画面描画を挟んでUIフリーズを軽減します。
- **アクセシビリティ**: キーボード操作時に現在位置が分かるよう、主要なボタン、入力欄、選択欄に `:focus-visible` のフォーカスリングを用意しています。


## UI導線

操作順は「画像選択 → 選択画像確認 → 選択画像の情報 → サイズ設定 → 位置設定 → プレビュー → ミニ調整 → 保存設定」です。1枚変換では、リサイズ / トリミングを選び、画像選択後にサイズ・位置・プレビューを確認して保存します。一括変換では、複数画像選択、共通のサイズ・位置設定、前へ / 次へでの代表画像確認、ZIP保存または連続ダウンロードの順に操作します。

「選択画像」は、加工設定前の元画像または作業中画像をざっと確認するための簡易表示です。位置調整、ドラッグ調整、トリミング枠、枠外暗表示、保存設定は反映しません。一方で「プレビュー」は、現在のサイズ設定、位置設定、保存設定を反映し、トリミング時は元画像全体、トリミング枠、枠外暗表示、余白方向を確認できます。

位置プリセットは「位置設定」パートに表示します。ミニ位置調整はプレビューを見ながら使う矢印・微調整専用UIです。ミニサイズ設定のプリセットは、サイズ設定パートで現在選択できるプリセットと同じ一覧を使います。元画像比率モード、トリミング方向、縦横比指定でプリセットがフィルタされた場合も、メインとミニで同じ内容になります。

成功メッセージ（画像読み込み、保存、ZIP保存、連続ダウンロード完了など）は数秒後に自動で消えます。エラーや警告メッセージは、確認しやすいように自動消去しない扱いを基本にしています。

## 1枚変換と一括変換の違い

### 1枚変換

1枚の画像を対象に、プレビュー直下へ集約した位置プリセット、矢印・微調整ボタンで位置を調整できます。プレビュー上のドラッグ調整も利用できます。横方向だけ調整できる場面では左 / 中央 / 右、縦方向だけ調整できる場面では上 / 中央 / 下、上下左右トリミングでは9方向プリセットを表示します。単体トリミングの「上下左右トリミング」では9方向の位置設定を表示しますが、指定サイズによって実際に効く方向は異なる場合があります。処理結果は「この結果を使って続ける」で作業中画像として確定でき、作業中ファイル名も現在の変換結果に合わせて更新されます。最初に読み込んだ画像へ戻す「元画像に戻す」も使えます。

### 一括変換

複数画像に同じルールを適用します。一括変換でも、リサイズ + トリミング、リサイズ + 余白付与、一括トリミングでは位置設定プリセットを使えます。一括変換では画像ごとに縦横比が異なる可能性があるため、横方向と縦方向の位置設定を共通設定として用意しています。横方向に調整が必要な画像では「左 / 中央 / 右」が効き、縦方向に調整が必要な画像では「上 / 中央 / 下」が効きます。一方で、画像ごとに縦横比やサイズが異なるため、フリー調整、矢印・微調整ボタン、プレビュー上のドラッグ調整、元画像比率フィルタ、上下のみ / 左右のみトリミング、縦横比入力補助などの詳細機能は使いません。

## リサイズモードとトリミングモード

### リサイズモード

指定した幅・高さに変換します。1枚変換では以下の4方式を選べます。

1. **元画像の縦横比を使用**: 元画像比率を維持します。片方のサイズを入力するともう片方を自動計算します。この方式では元画像と縦横比が合うプリセットだけを表示します。
2. **変形**: 指定した幅・高さへそのまま伸縮します。縦横比が変わる場合は歪みの注意を表示します。
3. **トリミング**: 元画像比率を維持して指定サイズを埋め、はみ出た部分を切ります。
4. **余白付与**: 元画像全体を残して指定サイズ内へ収め、不足部分に余白を付けます。

一括リサイズでは、画像ごとに元画像比率が異なるため「元画像の縦横比を使用」は使わず、変形 / トリミング / 余白付与のみを提供します。一括トリミング時はこのリサイズ処理方式を使わないため、変形 / トリミング / 余白付与のUIは表示しません。

### トリミングモード

画像を拡大縮小せず、切り抜きまたは余白付与で出力範囲を作ります。指定サイズが元画像より小さい方向はトリミングし、大きい方向は余白を付与します。

1枚変換では、上下左右 / 上下のみ / 左右のみの方向指定と、元画像比率・1:1・4:3・16:9 などの縦横比指定を利用できます。一括トリミングでは簡略化し、出力サイズ、位置、余白色だけを共通設定として使います。


## プレビュー近くの調整UI

- **位置プリセット**: 1枚変換では「位置設定」パートに表示します。横方向のみなら左 / 中央 / 右、縦方向のみなら上 / 中央 / 下、上下左右トリミングでは9方向から選べます。
- **矢印・微調整ボタン**: 1枚変換のプレビュー直下にだけ表示します。一括変換では表示しません。
- **ドラッグ調整**: 1枚変換のプレビュー上で、位置調整が可能な場合だけ利用できます。横方向だけ有効な場合は x だけ、縦方向だけ有効な場合は y だけ、上下左右で有効な場合は x/y の両方を変更します。プレビュー上の表示サイズに応じて移動量を計算し、ドラッグ可能時だけ `touch-action: none` を有効化するため、不要な場面ではスマホのページスクロールを妨げません。一括変換では利用しません。
- **サイズミニ設定**: 画像読み込み後はプレビュー直下で幅・高さ入力と、サイズ設定パートで現在選択できるものと完全一致したプリセットを使えます。1枚変換では、プレビュー近くにミニ位置調整とサイズミニ設定を配置しています。
- **フィルタ後のミニプリセット**: 「元画像の縦横比を使用」、トリミング方向、縦横比指定などでメインプリセットが絞り込まれた場合、ミニプリセットも同じ絞り込み結果を表示します。
- **固定サイズ項目の編集制御**: トリミング方向や縦横比指定によって固定されるサイズ項目は、メイン入力欄とミニ入力欄の両方で編集不可になります。

## トリミングプレビュー

プレビューcanvasは表示対象の縦横比に合わせて高さを自動調整します。トリミング枠プレビューでは元画像比率、最終結果プレビューでは出力サイズ比率を基準にし、スマホで縦長になりすぎないよう最小・最大高さを設けています。リサイズモードの「トリミング」とトリミングモードでは、元画像全体をプレビューに表示し、実際に残る範囲を枠で確認できます。枠外は半透明で暗く表示するため、位置プリセット、矢印・微調整ボタン、ドラッグ調整で切り抜き位置を確認しながら調整できます。一括変換のトリミング系プレビューでも、表示確認用として元画像全体と残る範囲の枠を表示します。

トリミングモードで指定サイズが元画像より大きく、出力画像に余白付与が発生する場合は、元画像上で実際に使われる範囲を枠で示し、プレビュー内に「余白あり：左右」「余白あり：上下」「余白あり：上下左右」のように余白方向を表示します。出力キャンバス比率の枠と余白部分の薄い表示により、位置設定で画像がどちらへ寄るかも確認しやすくしています。プレビュー表示用の高さ調整や枠表示は確認用であり、出力画像には含まれません。出力画像に余白が入るのは、余白付与モードや指定サイズが元画像より大きい場合など、仕様上必要な場合のみです。余白付与、変形、元画像比率の各方式では、従来どおり保存される最終結果をプレビュー表示します。

## 一括変換の保存方式

- **ZIP保存**: 最大30枚まで。ZIP生成には `libs/jszip.min.js` を使用します。ブラウザ内で動作する軽量な簡易ZIP生成処理で、画像データは非圧縮STORE方式で格納します。
- **連続ダウンロード**: 最大10枚まで。500ms 間隔で1枚ずつ保存します。ブラウザによっては複数ダウンロードがブロックされる場合があります。

一括変換、ZIPへのファイル追加、連続ダウンロードのループでは、各画像の処理後や重いBlob生成の前後にブラウザへ制御を返すタイミングを入れています。ZIP保存中は「変換中... 1 / n」「ZIP作成中... n / n (67%)」、連続ダウンロード中は「変換中... 1 / n」「連続ダウンロード中... 1 / n」のように進捗を表示します。ZIP生成内部でもファイルごと、およびCRC-32計算のチャンクごとにブラウザへ制御を返し、進捗percentは画像データの総バイト数に近い基準で計算します。大きい画像を多数ZIP化する場合、端末性能や空きメモリによって完了まで時間がかかる場合があります。Web Worker 化はしておらず、Vanilla HTML/CSS/JavaScriptのまま軽量な構成を維持しています。

ZIP内またはダウンロード時のファイル名は、リサイズでは `元ファイル名_幅x高さ.拡張子`、一括トリミングを含むトリミングでは `元ファイル名_crop_幅x高さ.拡張子` を基本にしています。


## 保存先について

保存先はブラウザの通常ダウンロード先です。Android Chromeでは、多くの場合「Download」または「ダウンロード」フォルダに保存されます。Web/PWA版では、アプリ側から任意の保存先を直接指定することはできません。

## 透明背景とJPEG

余白色は透明 / 白 / 黒から選べます。ただし JPEG は透明を保持できないため、透明指定時も白背景として保存されます。

## サイズと枚数の上限

- 読み込み画像の最大辺: 4624px（4096px級の画像対応を維持しつつ、3468×4624 / 4624×3468 などのスマホ写真も扱えます）
- 出力幅: 1〜4624px
- 出力高さ: 1〜4624px
- ZIP保存: 最大30枚
- 連続ダウンロード: 最大10枚

詳細プリセットにはスマホ写真向けに `3468×4624`（縦）と `4624×3468`（横）を追加しています。大きい画像や大きい出力サイズは、端末によって読み込みや保存に時間がかかる場合があります。

## 設定保存

画像そのものは保存せず、`localStorage` に以下の設定を分けて保存します。不正な保存値や範囲外の値が残っている場合は、読み込み時にデフォルトへ戻します。

- `commonSettings`: 保存形式、品質、余白色
- `singleSettings`: 1枚変換のモード、処理方式、サイズ、位置
- `batchSettings`: 一括変換のモード、処理方式、サイズ、位置

## 保存中の操作

1枚保存、「この結果を使って続ける」、ZIP保存、連続ダウンロードの処理中は、関連する保存ボタン、ナビゲーションボタン、画像選択入力、保存形式、サイズ設定などをまとめて無効化して二重実行や処理中の設定変更を防ぎます。処理中は「保存中...」「変換中...」「ZIP作成中...」「連続ダウンロード中...」のように状態を表示します。エラー時にも無効化状態が残らないよう、処理終了時に元の状態へ戻します。

## PWA更新時の注意

新しいService Workerが見つかり、すでに古いService Workerでページが制御されている場合は、画面下部に「新しいバージョンがあります。」という更新通知を表示します。初回インストール時には更新通知を出さず、既存ユーザーが新しいバージョンへ切り替える場面だけ通知します。

ユーザーが「更新する」を押すと、新しいService Workerへ切り替えた後にページを再読み込みします。「あとで」を選んだ場合は、次回アクセス時または再読み込み時に更新通知が再表示される場合があります。

更新通知で反映されない場合や古い画面が出続ける場合は、ブラウザキャッシュの削除、または PWA の再インストールを試してください。アセット構成や処理を変更した場合は `service-worker.js` の `CACHE_NAME` を更新してください。

## キーボード操作とフォーカス表示

Tabキーや外部キーボード、アクセシビリティ機能で操作する場合に現在のフォーカス位置が分かるよう、`button`、`input`、`select`、ファイル選択ボタン、プリセットボタン、主要アクションボタンへ `:focus-visible` ベースのフォーカスリングを追加しています。タッチ操作時に過剰な枠が出にくいよう、基本は `:focus` ではなく `:focus-visible` を使用しています。ダークモードでも見えるようにテーマカラーを利用します。
