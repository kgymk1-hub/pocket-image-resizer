# Pocket Image Resizer

Pocket Image Resizer は、スマホで画像を選び、指定したサイズへすばやくリサイズして保存する超軽量Web/PWAアプリです。Vanilla HTML / CSS / JavaScript のみで動作し、Android Chrome での利用とホーム画面追加を優先しています。

## 主な機能

- PNG / JPEG / WebP 画像の読み込み
- 画像プレビュー、元画像サイズ、元ファイル名、元ファイルサイズ、出力予定情報の表示
- 幅・高さの手入力（1〜4096px、整数のみ）
- よく使うサイズとカテゴリ別の出力サイズプリセット
- 詳細プリセットの元画像比率フィルタ
- 縦横比固定入力
- 中央トリミング / 余白あり
- トリミング比率プリセット
- 背景色：透明 / 白 / 黒
- PNG / JPEG / WebP 保存
- JPEG / WebP の品質設定
- 192×192 / 512×512 / 1024×1024 のアイコンセット保存
- localStorage による設定保存（画像そのものは保存しません）
- manifest と service worker による PWA 対応
- ダークモード対応

## 使い方

### 通常保存

1. 「画像を選択」から PNG / JPEG / WebP を読み込みます。
2. クイック設定で幅・高さを入力するか、よく使うプリセットを選びます。
3. 必要に応じて詳細プリセット、リサイズ方法、保存形式を調整します。
4. 「このサイズで保存」を押すと、現在の幅・高さで保存します。

保存名は `元画像名_幅x高さ.拡張子` です。例：`sample_512x512.png`。

### アイコンセット保存

「アイコンセット保存」は、現在選択中の保存形式、リサイズモード、背景色、トリミング比率設定を使い、以下の3サイズを順番に保存します。

- 192×192
- 512×512
- 1024×1024

複数ファイルのダウンロードはブラウザにブロックされる場合があります。その場合はブラウザ側で複数ダウンロードを許可するか、個別に保存してください。

## 保存先について

Web/PWA版では、保存先はブラウザの通常ダウンロード先になります。Android Chromeでは、多くの場合 Download または ダウンロード フォルダに保存されます。アプリ側から任意の保存先フォルダを直接指定することはできません。

## UI説明

- **クイック設定**：幅・高さ、縦横比固定、よく使うプリセット、現在設定表示、保存ボタン、アイコンセット保存をまとめています。
- **詳細プリセット**：大量のプリセットをカテゴリ別の折りたたみ表示にしています。
- **リサイズ方法**：中央トリミングと余白ありを切り替えます。余白ありでは背景色を余白に使います。
- **保存形式**：PNG / JPEG / WebP と、JPEG / WebP の品質を選べます。
- **トリミング比率プリセット**：中央トリミング時だけ使えます。サイズ指定ではなく切り抜き比率の指定です。
- **現在設定表示**：保存ボタン付近に、現在の保存形式とリサイズ方法を表示します。

## 出力サイズプリセット

プリセットは以下のカテゴリに分けています。

- よく使う：192×192、512×512、1024×1024、1280×720、1920×1080
- アイコン：favicon、PWA、Android、Apple touch icon など
- SNS：プロフィール画像、正方形投稿、ストーリーズ、OGP など
- 動画：360p、720p、1080p、4K など
- ブログ・Web：サムネイル、メイン画像など
- スマホ壁紙：縦長の壁紙目安
- カスタムに近い比率：4:3、3:2 などの汎用比率

大量プリセットは「詳細プリセットを表示」を開き、カテゴリをさらに開いて選択します。よく使うプリセットは常時表示しているため、頻度の高いサイズは少ないスクロールで選べます。

## 詳細プリセットの元画像比率フィルタ

「詳細プリセットを元画像に近い比率だけ表示」をONにすると、詳細プリセットだけを元画像の縦横比に近いものへ絞り込みます。よく使うプリセットは常に表示されます。

写真の縦横比に近いサイズだけを探したいときに使います。該当プリセットがない場合は、フィルタをOFFに戻すか、幅・高さを手入力してください。

## トリミング比率プリセット

トリミング比率プリセットは、出力サイズではなく「切り抜く比率」を指定する機能です。

- 中央トリミング時だけ有効です。
- 選択した比率で元画像の中央を切り抜きます。
- 切り抜き後に、現在指定している出力サイズへ変換します。
- 比率プリセットは切り抜き比率だけを指定します。出力サイズの比率が違うと画像が引き伸ばされます。

余白ありモードでは、元画像全体を収める仕様のためトリミング比率プリセットは使用できません。

## 保存形式

- **PNG**：透明背景を保持したい場合に向いています。
- **JPEG**：写真向けです。JPEGでは透明部分が背景色で塗られます。透明指定時は白になります。
- **WebP**：対応ブラウザでは透明背景と品質指定を利用できます。保存可否はブラウザ対応状況に依存します。

ブラウザが選択した保存形式に対応していない場合は、PNGで保存してください。

## PWAとして使う方法

1. ローカルサーバーまたはHTTPS環境でアプリを開きます。
2. Android Chrome のメニューから「ホーム画面に追加」を選びます。
3. 追加したアイコンから起動すると、スタンドアロン表示で使えます。

`file://` で直接開いた場合、ブラウザ仕様により service worker は登録されません。PWA確認時はローカルサーバーを使ってください。

## PWAアイコン

PWAアイコンは以下のファイルを使います。

```text
icons/icon.svg
icons/icon-192.png
icons/icon-512.png
```

アップロード済み画像から `icons/icon-192.png` と `icons/icon-512.png` を作成・配置済みです。`manifest.json` は SVG アイコンと PNG アイコンの両方を参照しています。

PNGアイコンを差し替えた場合は、`service-worker.js` の `CACHE_NAME` を変更してください。キャッシュが残っていると、古いアイコンや古いアプリ内容が表示される場合があります。

## PWA更新が反映されない場合

1. `service-worker.js` の `CACHE_NAME` を変更する
2. Chromeで該当サイトのキャッシュを削除する
3. ホーム画面に追加済みのPWAを一度削除する
4. もう一度ブラウザで開き、ホーム画面に追加し直す

## ローカルサーバー起動例

```bash
python -m http.server 8000
```

ブラウザで以下を開きます。

```text
http://localhost:8000/
```

## 既知の制限

- EXIF回転未対応です。
- HEIC未対応です。
- 画像履歴保存はありません。
- ZIP生成はありません。
- 複数ダウンロードはブラウザにブロックされる場合があります。
- WebP保存はブラウザ対応状況に依存します。
- Web/PWA版では保存先フォルダをアプリ側で直接指定できません。
- 手動でトリミング位置を指定する機能はありません。
- 明るさ・色調補正、フィルター加工、文字入れ、クラウド保存、既存ファイルへの上書き保存には対応しません。
