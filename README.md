# Pocket Image Resizer

Pocket Image Resizer は、スマホで画像を選び、指定サイズへすばやくリサイズして保存する超軽量Webアプリです。Vanilla HTML / CSS / JavaScript だけで動作し、PWAとしてAndroid Chromeのホーム画面から使うことを重視しています。

## 機能一覧

- PNG / JPEG / WebP の読み込み
- プレビュー、元画像サイズ、出力予定サイズ、元ファイル名、元ファイルサイズの表示
- 192×192 / 512×512 / 1024×1024 / 1280×720 / 1920×1080 プリセット
- 幅・高さの手入力（1〜4096px、整数のみ）
- 縦横比固定（初期ON）
- 中央トリミング（cover）
- 余白あり（contain）
- 余白背景：透明 / 白 / 黒
- PNG / JPEG / WebP 保存
- JPEG / WebP の品質設定（低 0.6、標準 0.85、高 0.95）
- 現在サイズの個別保存
- 192 / 512 / 1024px のアイコンセット保存
- localStorage による設定保存（画像自体は保存しません）
- manifest と service worker によるPWA対応
- ダークモード対応

## 使い方

1. `index.html` をブラウザで開く、またはローカルサーバーで配信します。
2. 「画像を選択」から PNG / JPEG / WebP を選びます。
3. プリセット、または幅・高さ入力で出力サイズを指定します。
4. 必要に応じて「縦横比を固定」を切り替えます。
5. 「中央トリミング」または「余白あり」を選びます。
6. 保存形式を PNG / JPEG / WebP から選びます。
7. 「このサイズで保存」または「アイコンセット保存」を押します。

保存名は `元画像名_幅x高さ.拡張子` です。例：`icon_512x512.png`、`photo_1920x1080.jpg`、`logo_512x512.webp`。

## PWAとして使う方法

1. ローカルサーバーまたはHTTPS環境でアプリを開きます。
2. Android Chrome のメニューから「ホーム画面に追加」を選びます。
3. 追加したアイコンから起動すると、スタンドアロン表示で使えます。

`file://` で直接開いた場合、ブラウザ仕様により service worker は登録されません。PWA確認時はローカルサーバーを使ってください。

## service worker更新時の注意

`service-worker.js` は主要ファイルをオフラインキャッシュします。アプリのHTML / CSS / JavaScript / manifest / icon を更新した時は、`CACHE_NAME` を変更してください。キャッシュ名を変えることで、古いキャッシュを破棄しやすくなります。

## 複数ダウンロード時の注意

「アイコンセット保存」はZIP化せず、192×192、512×512、1024×1024 の3ファイルを順番にダウンロードします。ブラウザ設定や環境によっては、複数ダウンロードがブロックされる可能性があります。その場合はブラウザの許可設定を確認するか、個別に保存してください。

## 対応形式

| 用途 | 形式 |
| --- | --- |
| 読み込み | PNG / JPEG / WebP |
| 保存 | PNG / JPEG / WebP |

PNG と WebP は透明背景を保存できます。JPEG は透明を保持できないため、透明背景が選ばれている場合も白背景として保存します。

## 今後の改善候補

- スマホ写真のEXIF回転情報によって、端末やブラウザによっては向きが期待と異なる可能性があります。将来的な改善候補としてEXIF回転対応があります。
- 端末性能に応じた大きな画像のメモリ使用量対策
- ブラウザ別の複数ダウンロード挙動の案内強化
- インストール済みPWAでの更新通知

## 開発方針

- Vanilla HTML / CSS / JavaScript のまま保つ
- npm、ビルド環境、外部画像ライブラリを追加しない
- 画像編集アプリではなく、画像リサイズ専用アプリとしてシンプルに保つ
- Android Chrome と片手操作しやすいスマホUIを優先する
- コメントは「なぜ必要か」「計算の注意点」がある箇所に絞る
