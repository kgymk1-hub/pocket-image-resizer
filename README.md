# Pocket Image Resizer

Pocket Image Resizer は、Android Chrome などスマホで使いやすいことを優先した Vanilla HTML / CSS / JavaScript 製の軽量 PWA です。React / Vue / Svelte、npm、ビルド環境は使っていません。

## 主な特徴

- **1枚変換**: 1枚の画像をプレビューしながら細かく調整して保存します。
- **一括変換**: 複数画像へ共通設定を適用し、ZIP または連続ダウンロードで保存します。
- **保存形式**: PNG / JPEG / WebP に対応します。JPEG / WebP は品質（低 0.6、標準 0.85、高 0.95）を選べます。PNG では品質設定が不要なため、品質UIは非表示または無効化されます。
- **PWA対応**: `manifest.json`、`service-worker.js`、アイコン一式を維持しています。

## 1枚変換と一括変換の違い

### 1枚変換

1枚の画像を対象に、プレビュー上のドラッグや矢印ボタンで位置を調整できます。横方向だけ調整できる場面では左右のみ、縦方向だけ調整できる場面では上下のみを表示します。処理結果は「この結果を使って続ける」で作業中画像として確定でき、最初に読み込んだ画像へ戻す「元画像に戻す」も使えます。

### 一括変換

複数画像に同じルールを適用します。画像ごとに縦横比やサイズが異なるため、矢印ボタンや微調整ボタンを含むフリー位置調整、元画像比率フィルタ、上下のみ / 左右のみトリミング、縦横比入力補助などの詳細機能は使いません。

## リサイズモードとトリミングモード

### リサイズモード

指定した幅・高さに変換します。1枚変換では以下の4方式を選べます。

1. **元画像の縦横比を使用**: 元画像比率を維持します。片方のサイズを入力するともう片方を自動計算します。この方式では元画像と縦横比が合うプリセットだけを表示します。
2. **変形**: 指定した幅・高さへそのまま伸縮します。縦横比が変わる場合は歪みの注意を表示します。
3. **トリミング**: 元画像比率を維持して指定サイズを埋め、はみ出た部分を切ります。
4. **余白付与**: 元画像全体を残して指定サイズ内へ収め、不足部分に余白を付けます。

一括リサイズでは、画像ごとに元画像比率が異なるため「元画像の縦横比を使用」は使わず、変形 / トリミング / 余白付与のみを提供します。

### トリミングモード

画像を拡大縮小せず、切り抜きまたは余白付与で出力範囲を作ります。指定サイズが元画像より小さい方向はトリミングし、大きい方向は余白を付与します。

1枚変換では、上下左右 / 上下のみ / 左右のみの方向指定と、元画像比率・1:1・4:3・16:9 などの縦横比指定を利用できます。一括トリミングでは簡略化し、出力サイズ、位置、余白色だけを共通設定として使います。

## 一括変換の保存方式

- **ZIP保存**: 最大30枚まで。ZIP生成には `libs/jszip.min.js` を使用します。
- **連続ダウンロード**: 最大10枚まで。500ms 間隔で1枚ずつ保存します。ブラウザによっては複数ダウンロードがブロックされる場合があります。

ZIP内またはダウンロード時のファイル名は、リサイズでは `元ファイル名_幅x高さ.拡張子`、一括トリミングを含むトリミングでは `元ファイル名_crop_幅x高さ.拡張子` を基本にしています。

## 透明背景とJPEG

余白色は透明 / 白 / 黒から選べます。ただし JPEG は透明を保持できないため、透明指定時も白背景として保存されます。

## サイズと枚数の上限

- 出力幅: 1〜4096px
- 出力高さ: 1〜4096px
- ZIP保存: 最大30枚
- 連続ダウンロード: 最大10枚

## 設定保存

画像そのものは保存せず、`localStorage` に以下の設定を分けて保存します。不正な保存値や範囲外の値が残っている場合は、読み込み時にデフォルトへ戻します。

- `commonSettings`: 保存形式、品質、余白色
- `singleSettings`: 1枚変換のモード、処理方式、サイズ、位置
- `batchSettings`: 一括変換のモード、処理方式、サイズ、位置、保存方式

## 保存中の操作

1枚保存、「この結果を使って続ける」、ZIP保存、連続ダウンロードの処理中は、対象ボタンを無効化して二重実行を防ぎます。処理中は「保存中...」「変換中...」「ZIP作成中...」のように状態を表示します。

## PWA更新時の注意

Service Worker が旧キャッシュを保持している場合、更新後に再読み込み、キャッシュ削除、または PWA の再インストールが必要になることがあります。アセット構成や処理を変更した場合は `service-worker.js` の `CACHE_NAME` を更新してください。
