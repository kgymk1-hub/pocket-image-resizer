# Pocket Image Resizer

Pocket Image Resizer は、Android Chrome などスマホで使いやすいことを優先した Vanilla HTML / CSS / JavaScript 製の軽量 PWA です。React / Vue / Svelte、npm、ビルド環境は使っていません。

## 主な特徴

- **1枚変換**: 1枚の画像をプレビューしながら細かく調整して保存します。
- **一括変換**: 複数画像へ共通設定を適用し、ZIP または連続ダウンロードで保存します。
- **保存形式**: PNG / JPEG / WebP に対応します。JPEG / WebP は品質（低 0.6、標準 0.85、高 0.95）を選べます。PNG では品質設定が不要なため、品質UIは非表示または無効化されます。
- **PWA対応**: `manifest.json`、`service-worker.js`、アイコン一式を維持しています。

## 1枚変換と一括変換の違い

### 1枚変換

1枚の画像を対象に、プレビュー直下へ集約した位置プリセット、矢印・微調整ボタンで位置を調整できます。プレビュー上のドラッグ調整も利用できます。横方向だけ調整できる場面では左 / 中央 / 右、縦方向だけ調整できる場面では上 / 中央 / 下、上下左右トリミングでは9方向プリセットを表示します。単体トリミングの「上下左右トリミング」では9方向の位置設定を表示しますが、指定サイズによって実際に効く方向は異なる場合があります。処理結果は「この結果を使って続ける」で作業中画像として確定でき、作業中ファイル名も現在の変換結果に合わせて更新されます。最初に読み込んだ画像へ戻す「元画像に戻す」も使えます。

### 一括変換

複数画像に同じルールを適用します。一括変換でも、リサイズ + トリミング、リサイズ + 余白付与、一括トリミングでは位置設定プリセットを使えます。一括変換では画像ごとに縦横比が異なる可能性があるため、横方向と縦方向の位置設定を共通設定として用意しています。横方向に調整が必要な画像では「左 / 中央 / 右」が効き、縦方向に調整が必要な画像では「上 / 中央 / 下」が効きます。一方で、画像ごとに縦横比やサイズが異なるため、フリー調整、矢印・微調整ボタン、プレビュー上のドラッグ調整、元画像比率フィルタ、上下のみ / 左右のみトリミング、縦横比入力補助などの詳細機能は使いません。

## リサイズモードとトリミングモード

### リサイズモード

指定した幅・高さに変換します。1枚変換では以下の4方式を選べます。

1. **元画像の縦横比を使用**: 元画像比率を維持します。片方のサイズを入力するともう片方を自動計算します。この方式では元画像と縦横比が合うプリセットだけを表示します。
2. **変形**: 指定した幅・高さへそのまま伸縮します。縦横比が変わる場合は歪みの注意を表示します。
3. **トリミング**: 元画像比率を維持して指定サイズを埋め、はみ出た部分を切ります。
4. **余白付与**: 元画像全体を残して指定サイズ内へ収め、不足部分に余白を付けます。

一括リサイズでは、画像ごとに元画像比率が異なるため「元画像の縦横比を使用」は使わず、変形 / トリミング / 余白付与のみを提供します。一括トリミング時はこのリサイズ処理方式を使わないため、変形 / トリミング / 余白付与のUIは表示しません。

### トリミングモード

画像を拡大縮小せず、切り抜きまたは余白付与で出力範囲を作ります。指定サイズが元画像より小さい方向はトリミングし、大きい方向は余白を付与します。

1枚変換では、上下左右 / 上下のみ / 左右のみの方向指定と、元画像比率・1:1・4:3・16:9 などの縦横比指定を利用できます。一括トリミングでは簡略化し、出力サイズ、位置、余白色だけを共通設定として使います。


## プレビュー近くの調整UI

- **位置プリセット**: 1枚変換ではプレビュー直下へ集約しています。横方向のみなら左 / 中央 / 右、縦方向のみなら上 / 中央 / 下、上下左右トリミングでは9方向から選べます。
- **矢印・微調整ボタン**: 1枚変換のプレビュー直下にだけ表示します。一括変換では表示しません。
- **ドラッグ調整**: 1枚変換のプレビュー上で利用できます。横方向だけ有効な場合は x だけ、縦方向だけ有効な場合は y だけ、上下左右で有効な場合は x/y の両方を変更します。一括変換では利用しません。
- **サイズミニ設定**: 画像読み込み後はプレビュー直下で幅・高さ入力と 192×192 / 512×512 / 1024×1024 / 1280×720 / 1920×1080 の主要プリセットを使えます。詳細プリセットは従来どおり設定カード内にも残しています。1枚変換では、プレビュー近くに位置調整とサイズミニ設定を配置しています。プレビューを見ながら、幅・高さや主要プリセット、位置調整、微調整を行えます。
- **元画像比率モードのミニプリセット**: 「元画像の縦横比を使用」では、メインプリセットだけでなくプレビュー近くのミニプリセットも、元画像比率に合うものだけ表示します。
- **固定サイズ項目の編集制御**: トリミング方向や縦横比指定によって固定されるサイズ項目は、メイン入力欄とミニ入力欄の両方で編集不可になります。

## トリミングプレビュー

プレビューcanvasは表示対象の縦横比に合わせて高さを自動調整します。トリミング枠プレビューでは元画像比率、最終結果プレビューでは出力サイズ比率を基準にし、スマホで縦長になりすぎないよう最小・最大高さを設けています。リサイズモードの「トリミング」とトリミングモードでは、元画像全体をプレビューに表示し、実際に残る範囲を枠で確認できます。枠外は半透明で暗く表示するため、位置プリセット、矢印・微調整ボタン、ドラッグ調整で切り抜き位置を確認しながら調整できます。一括変換のトリミング系プレビューでも、表示確認用として元画像全体と残る範囲の枠を表示します。

トリミングモードで指定サイズが元画像より大きく、出力画像に余白付与が発生する場合は、元画像上で実際に使われる範囲を枠で示し、プレビュー内に「余白あり：左右」「余白あり：上下」「余白あり：上下左右」のように余白方向を表示します。出力キャンバス比率の枠と余白部分の薄い表示により、位置設定で画像がどちらへ寄るかも確認しやすくしています。プレビュー表示用の高さ調整や枠表示は確認用であり、出力画像には含まれません。出力画像に余白が入るのは、余白付与モードや指定サイズが元画像より大きい場合など、仕様上必要な場合のみです。余白付与、変形、元画像比率の各方式では、従来どおり保存される最終結果をプレビュー表示します。

## 一括変換の保存方式

- **ZIP保存**: 最大30枚まで。ZIP生成には `libs/jszip.min.js` を使用します。
- **連続ダウンロード**: 最大10枚まで。500ms 間隔で1枚ずつ保存します。ブラウザによっては複数ダウンロードがブロックされる場合があります。

ZIP内またはダウンロード時のファイル名は、リサイズでは `元ファイル名_幅x高さ.拡張子`、一括トリミングを含むトリミングでは `元ファイル名_crop_幅x高さ.拡張子` を基本にしています。


## 保存先について

保存先はブラウザの通常ダウンロード先です。Android Chromeでは、多くの場合「Download」または「ダウンロード」フォルダに保存されます。Web/PWA版では、アプリ側から任意の保存先を直接指定することはできません。

## 透明背景とJPEG

余白色は透明 / 白 / 黒から選べます。ただし JPEG は透明を保持できないため、透明指定時も白背景として保存されます。

## サイズと枚数の上限

- 出力幅: 1〜4096px
- 出力高さ: 1〜4096px
- ZIP保存: 最大30枚
- 連続ダウンロード: 最大10枚

## 設定保存

画像そのものは保存せず、`localStorage` に以下の設定を分けて保存します。不正な保存値や範囲外の値が残っている場合は、読み込み時にデフォルトへ戻します。

- `commonSettings`: 保存形式、品質、余白色
- `singleSettings`: 1枚変換のモード、処理方式、サイズ、位置
- `batchSettings`: 一括変換のモード、処理方式、サイズ、位置

## 保存中の操作

1枚保存、「この結果を使って続ける」、ZIP保存、連続ダウンロードの処理中は、関連する保存ボタン、ナビゲーションボタン、画像選択入力をまとめて無効化して二重実行を防ぎます。処理中は「保存中...」「変換中...」「ZIP作成中...」のように状態を表示します。

## PWA更新時の注意

PWAで古い画面が表示される場合は、ブラウザキャッシュ削除またはPWA再インストールが必要です。Service Worker が旧キャッシュを保持している場合、更新後に再読み込み、キャッシュ削除、または PWA の再インストールが必要になることがあります。アセット構成や処理を変更した場合は `service-worker.js` の `CACHE_NAME` を更新してください。
