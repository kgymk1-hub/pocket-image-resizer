# Pocket Image Resizer

Pocket Image Resizer は、スマホで画像を選び、指定サイズへすばやくリサイズして保存する超軽量Webアプリです。Vanilla HTML / CSS / JavaScript だけで動作し、Android Chrome と PWA 利用を優先しています。

## 機能一覧

- PNG / JPEG / WebP の読み込みと保存
- プレビュー、元画像サイズ、元ファイル名、元ファイルサイズ、出力予定サイズの表示
- 幅・高さの手入力（1〜4096px、整数のみ）
- 縦横比固定入力
- カテゴリ別の出力サイズプリセット
- 元画像比率に近いプリセットだけ表示するフィルタ
- 中央トリミング / 余白あり
- トリミング比率プリセット
- 余白背景：透明 / 白 / 黒
- JPEG / WebP の品質設定
- 192×192 / 512×512 / 1024×1024 のアイコンセット保存
- localStorage による設定保存（画像そのものは保存しません）
- manifest と service worker による PWA 対応
- ダークモード対応

## 出力サイズプリセット

出力サイズプリセットは、よく使う、アイコン、SNS、動画、ブログ・Web、スマホ壁紙、カスタムに近い比率に分けて表示します。各プリセットにはサイズと用途を表示します。

例：

- 512×512 - PWAアイコン標準
- 1920×1080 - Full HD / 16:9
- 1080×1920 - ストーリーズ / 縦長

「元画像に近い比率のプリセットだけ表示」をONにすると、元画像の縦横比に近いプリセットだけを表示します。画像未選択時は全プリセットを表示します。該当プリセットがない場合は、フィルタをOFFにするかカスタムサイズを指定してください。

## トリミング比率プリセット機能

「トリミング比率プリセットを使う」をONにすると、元画像を指定比率で中央トリミングしてから、現在指定している出力サイズへリサイズします。

対応比率：

- 元画像の比率
- 1:1
- 4:3 / 3:4
- 3:2 / 2:3
- 16:9 / 9:16
- 21:9
- 5:4 / 4:5

この機能は中央トリミング時のみ有効です。余白ありモードでは使用できません。

## 中央トリミングと余白ありの違い

- 中央トリミング：出力サイズを完全に埋めます。必要に応じて元画像の上下または左右を中央基準で切り抜きます。
- 余白あり：元画像全体を出力サイズ内に収めます。余った部分は透明 / 白 / 黒の背景で埋めます。

## PNG / JPEG / WebP保存

PNG は透明背景に向いています。JPEG は透明を保持できないため、透明背景または未指定の場合は白背景として保存します。WebP は対応ブラウザで透明背景と品質指定を利用できます。ブラウザが選択形式の保存に対応していない場合は、PNGで保存してください。

保存名は `元画像名_幅x高さ.拡張子` です。例：`icon_512x512.png`、`photo_1920x1080.jpg`、`logo_512x512.webp`。

## アイコンセット保存

「アイコンセット保存」は、現在選択中の保存形式、リサイズモード、トリミング比率設定を使い、以下の3サイズを順番に保存します。

- 192×192
- 512×512
- 1024×1024

## 複数ダウンロード時の注意

アイコンセット保存はZIP化せず、3ファイルを順番にダウンロードします。ブラウザ設定や環境によっては複数ダウンロードがブロックされる可能性があります。その場合はブラウザの許可設定を確認するか、個別に保存してください。

## ローカルサーバーでの起動例

```bash
python -m http.server 8000
```

ブラウザで以下を開きます。

```text
http://localhost:8000/
```

## PWAとして使う方法

1. ローカルサーバーまたはHTTPS環境でアプリを開きます。
2. Android Chrome のメニューから「ホーム画面に追加」を選びます。
3. 追加したアイコンから起動すると、スタンドアロン表示で使えます。

`file://` で直接開いた場合、ブラウザ仕様により service worker は登録されません。通常のリサイズ機能は使えますが、PWA確認時はローカルサーバーを使ってください。

## service worker更新時の注意

`service-worker.js` は主要ファイルをオフラインキャッシュします。HTML / CSS / JavaScript / manifest / icon を更新した時は、`CACHE_NAME` を変更してください。キャッシュ名を変えることで、古いキャッシュを破棄しやすくなります。

## PWA更新が反映されない場合

1. `service-worker.js` の `CACHE_NAME` を変更する
2. Chromeで該当サイトのキャッシュを削除する
3. ホーム画面に追加済みのPWAを一度削除する
4. もう一度ブラウザで開き、ホーム画面に追加し直す

## 既知の制限

- EXIF回転補正は未対応です。スマホ写真では、端末やブラウザによって向きが期待と異なる場合があります。
- HEIC、ZIP生成、クラウド保存、画像履歴保存、既存ファイルへの上書き保存には対応しません。
- 手動でトリミング位置を動かす機能、明るさ・色調補正、フィルター加工、文字入れは実装していません。

## 開発方針

- Vanilla HTML / CSS / JavaScript のまま保つ
- React / Vue / Svelte、npm、ビルド環境、外部ライブラリを追加しない
- 画像編集アプリではなく、画像リサイズ専用アプリとしてシンプルに保つ
- Android Chrome と片手操作しやすいスマホUIを優先する
- 画像そのものは保存せず、設定のみ localStorage に保存する
- コメントは「なぜ必要か」「計算の注意点」がある箇所に絞る
